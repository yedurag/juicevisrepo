In this section I will try to explain how this web app was made. The explanation is brief and step by step. All the files are uploaded in the github repo given below

Steps in the Making

Step 1. Initial Planning and Data Study

In this step I came up with an initial plan for the project. About two days were spent in studying the data, figuring out the requirements and understanding the working of the Nutritionix API. The files- products.csv, fruits.csv and output1 in the github repo are actually the initial data files used for the data study. After that two python files were developed to clean the data. These are initialanalysis.py and ingredientsanalysis.py.

Step 2. Python Flask Phase

In this step I used Flask, SQL Alchemy and Postgres to develop a back end system to pull the data and inject it to the front end upon request. I used requests library for data pull. The following files can be found in the github repo-

app.py - The main flask app file

config.py - My configs for the app locally and in heroku

manage.py - For db migration

models.py - My sql alchemy db model

In this phase I heavily relied upon the Flask by Example tutorial that I did about 6 months back. It is a great tutorial by my favorite python teacher - Michael Herman. I am grateful to him.

Step 3. Front End Phase

By now I had a good understanding of the data and knew how the visualizations should look like. I drew the design on paper. The images can be found in the repo.

Once the design was complete, I went in and coded it using the dc.js, d3.js and crossfilter.js. Bootstrap was also used extensively to beautify the dashboard.

In the end I hosted the app in Heroku. While doing this I incorporated the redis task queue to make tha app fail safe.
